Accurate, produces moire-free results, but tends to produce blurrier images.

When upsampling, it is similar to nearest-neighbor.

◆ CUBIC

int CUBIC = 3
static

Bi-cubic interpolation.

Better quality than bi-linear, slower. Produces high-quality results in a larger range than that of bi-linear.

◆ LANCZOS4

int LANCZOS4 = 4
static

Lanczos (Sinc) interpolation over 8x8 neighborhood.

Produces very high quality results, but slower than bi-cubic. Retains sharp edges like those of text when downsampling.

◆ LINEAR

int LINEAR = 2
static

Bi-linear interpolation.

Better quality than nearest-neighbor, slower. Okay when the target size is not too different from the source size.

◆ NEAREST

Nearest-neighbor interpolation.

Lowest quality. Produces blocky images, especially when upsampling.

Fast. Otherwise, downscale the bitmap to match the dpi setting via the selected method before adding it to the PDF if the calculated image DPI after stretching is greater than dpi. Slower.
