enum scanbotsdk_credit_card_recognition_mode_t

scanbotsdk_credit_card_recognition_mode_t enum

The recognition mode

Enumerator
SCANBOTSDK_CREDIT_CARD_RECOGNITION_MODE_LIVE 

The recognizer will merge all information from multiple frames to provide the best possible result. Use this mode when the input is a video or camera stream.

SCANBOTSDK_CREDIT_CARD_RECOGNITION_MODE_SINGLE_SHOT 

The recognizer will only use the current frame and will spend additional time to ensure the best possible result. Use this mode when scanning single images, e.g. imported from the gallery.

Creates a new instance of scanbotsdk_credit_card_recognizer_configuration with given params and stores it in the location specified by the last argument.

As a rule if the function returns SCANBOTSDK_OK then it takes ownership of the pointers to scanbotsdk structures scanbotsdk_..._t* which otherwise would have to be freed by the caller by calling appropriate scanbotsdk_..._free and possibly invalidates them. The only exception is scanbotsdk_image_t* (if applicable) which is a long-lived object and should be freed by the caller. If the function returns an error code then the pointers to scanbotsdk structures remain valid and the caller is responsible for freeing them.

Parameters
use_document_detectorIf true, the document detector will be used to find where the credit card is in the input image. If false, the recognizer will assume that the credit card has been pre-cropped and takes the entirety of the input image. Default is true
recognition_modeThe recognition mode Default is LIVE
require_expiry_dateWhether the expiry date is required for a successful recognition Default is true
require_cardholder_nameWhether the cardholder name is required for a successful recognition Default is true
