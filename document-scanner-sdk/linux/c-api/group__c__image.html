Image Ref Path Load Mode

Enumerator
SCANBOTSDK_PATH_LOAD_MODE_EAGER 

Image is immediately loaded into memory.

SCANBOTSDK_PATH_LOAD_MODE_LAZY 

Image is loaded into memory the first time it's requested. Specified path must exist at the time of loading.

SCANBOTSDK_PATH_LOAD_MODE_LAZY_WITH_COPY 

Image is loaded into memory the first time it's requested. Specified path is copied into internal directory and so isn't required to exist at the time of loading.

SBSDK_API scanbotsdk_error_code_t scanbotsdk_basic_image_load_options_create ( scanbotsdk_rect_t roi,
scanbotsdk_basic_image_load_options_t ** result )

Creates a new instance of scanbotsdk_basic_image_load_options with given params and stores it in the location specified by the last argument.

As a rule if the function returns SCANBOTSDK_OK then it takes ownership of the pointers to scanbotsdk structures scanbotsdk_..._t* which otherwise would have to be freed by the caller by calling appropriate scanbotsdk_..._free and possibly invalidates them. The only exception is scanbotsdk_image_t* (if applicable) which is a long-lived object and should be freed by the caller. If the function returns an error code then the pointers to scanbotsdk structures remain valid and the caller is responsible for freeing them.
